# W6-01 — Hammerspoon `⌥Space` palette + local approval cards

**Status:** todo

**Flags:** user-assist
**Handoff refs:** §4 UI + IPC bildirim teslimi ⚑, §5 güvenlik #5 ⚑ + uygulama düzlemi ⚑, §7 TCC checklist ⚑

## Goal
After this task the user can press `⌥Space` anywhere on macOS to open a Kâhya command palette (Hammerspoon), submit a Turkish command, and get the answer back as a notification — and Kâhya's **local approval surface** moves from the CLI prompt to Hammerspoon cards (`hs.notify` with action buttons), including the typed-"onayla" W3 flow. Palette-open and first-token timestamps are recorded in the `events` table so the north-star latency metric becomes measurable.

## Deliverables
- `hammerspoon/kahya.lua` — palette (`hs.chooser` text input on `⌥Space`), approval cards (`hs.notify` + `hs.dialog`), `kahyaNotify` result notifications, `hs.ipc.cliInstall("/opt/homebrew")`; all user-visible strings Turkish.
- `Makefile` target `hammerspoon-install`: copies `hammerspoon/kahya.lua` → `~/.hammerspoon/kahya.lua`, substituting the `@KAHYA_BIN@` placeholder with the absolute path of the built `kahya` CLI (`$(abspath ...)` of W0-02's build output — `hs.task` cannot PATH-resolve), appends `require("kahya")` to `~/.hammerspoon/init.lua` if absent.
- kahyad (extend W12-01 UDS server + W3-02/W3-06 approval plumbing; adapt names to the existing package layout if it differs): `GET /approvals/pending`, `POST /approvals/{id}/decision` (fields: `approve` bool, `typed` string — **no** `surface` field: the server itself stamps `surface:"local"` on every decision arriving over the UDS; the surface label is channel-derived, never client-supplied, so no caller can forge the §5 #5 local-surface requirement); `palette_open` + `first_token` event writes; on new pending approval, exec the Hammerspoon CLI (`hs -c 'kahyaShowApproval(<id>)'`, path from config key `ui.hs_cli`, default `/opt/homebrew/bin/hs`).
- Generic local notification path (§4 IPC ⚑ "arka-plan görev sonuçları aynı kanaldan `trace_id` ile döner"): global `kahyaNotify(payloadJson)` in `kahya.lua` (`hs.notify`, title + first lines + `trace_id` shown) + kahyad helper that delivers **background/scheduled task results** locally via `ui.hs_cli`. If the exec fails, log a warning — result stays reachable via `kahya log --trace <id>` (delivery degradation, never loss).
- `kahyad/cmd/kahya/`: `kahya ask --palette-opened-at <unix-seconds-float>` flag; `kahya approvals list|show|decide` subcommands; `kahya debug emit-approval --class W2|W3` helper (refuses unless `KAHYA_ENV=dev`).
- `docs/tcc-w6-checklist.md` — the §7 checklist specialized for W6 with verification commands: Hammerspoon Accessibility/Microphone/Input Monitoring **plus the §7 row-3 Full Disk Access grant for kahyad** (the fs tool's owning process, W3-03) — grant instructions for launchd-started kahyad, a verification command exercising an `fs_read` of a TCC-protected directory, and blocked-user handling (until granted, the fs tool keeps failing cleanly with W3-03's `Tam Disk Erişimi gerekli: <path>`).
- Tests: `kahyad/internal/server/palette_events_test.go`, `kahyad/internal/policy/approval_surface_test.go` (names indicative — put them next to the code they test).

## Steps
1. `brew install --cask hammerspoon` if missing; enable "Launch Hammerspoon at login".
2. Write `hammerspoon/kahya.lua`: declare `local kahyaBin = "@KAHYA_BIN@"` at the top (Hammerspoon's GUI environment has no shell `PATH`; `hs.task` requires an absolute executable path — the `hammerspoon-install` target substitutes the placeholder). `hs.hotkey.bind({"alt"}, "space", ...)` opens an `hs.chooser` with placeholder `Kâhya'ya yaz…`. Capture `hs.timer.secondsSinceEpoch()` **at hotkey press** into `paletteOpenedAt`. On submit, run (via `hs.task`) `kahyaBin ask --palette-opened-at <paletteOpenedAt> -- <text>`; show completion via `hs.notify` (title `Kâhya`, informative text = first lines of the answer; full output stays reachable via `kahya log --trace <id>`).
3. In kahyad: when a task envelope carries `palette_opened_at`, insert an `events` row `kind='palette_open'` with that timestamp and the task `trace_id`; when the first stream token of model output is relayed for that task, insert `kind='first_token'`. Emit `first_token` at **every** point where kahyad first relays streamed model output: the W12-08 forward proxy (cloud + stub models) **and** the W3-08 secret-lane local path — otherwise secret-lane palette commands never log `first_token` and the north-star metric silently drops the local lane. Both rows JSONL-logged with `trace_id`.
4. Implement `GET /approvals/pending` and `POST /approvals/{id}/decision` on the UDS server, delegating verification (hash of approved bytes, NFC normalize, bidi/zero-width/homoglyph strip — W3-06) to the existing pipeline. A `decision` on a W3 approval is accepted **only** if `typed`, after NFC normalization, is byte-equal to `onayla`. Ledger every decision with `surface:"local"` — stamped by the server from the channel (UDS ⇒ local), never read from the request body; any `surface` key in the body is ignored (mirror of W3-07, where the Telegram code path stamps `remote`).
5. Add `kahya approvals list|show <id>|decide <id> (--approve --typed <s> | --reject)` to the CLI. `show` prints the byte-exact rendered diff.
6. Approval cards in `kahya.lua`: global function `kahyaShowApproval(id)` fetches `kahya approvals show <id> --json`, pops `hs.notify` (title `Kâhya — onay bekliyor`, action button `Görüntüle`). The button opens: for W1/W2 an `hs.dialog.blockAlert` with the diff and `Onayla`/`Reddet` buttons; for W3 an `hs.dialog.textPrompt` whose informative text is the diff and which instructs: `Onaylamak için "onayla" yazın`. The choice is sent back via `kahya approvals decide`.
7. Call `hs.ipc.cliInstall("/opt/homebrew")` in `kahya.lua` (Apple Silicon Homebrew prefix, matching the `ui.hs_cli` default `/opt/homebrew/bin/hs`; the parameterless default installs under `/usr/local`, which does not match this machine); wire kahyad's new-pending-approval hook to exec `ui.hs_cli`, and add the generic `kahyaNotify(payloadJson)` entry point + kahyad-side helper for background/scheduled task results (§4 ⚑ bildirim teslimi). If any exec fails, log a warning — the approval stays discoverable via `kahya approvals list` (fail-closed: nothing auto-approves), the result via `kahya log --trace <id>`.
8. Add the `hammerspoon-install` Makefile target and run it.
9. Write `docs/tcc-w6-checklist.md`: grant Hammerspoon Accessibility, Microphone, Input Monitoring in System Settings; set Hammerspoon's notification style to **Alerts** ("Uyarılar") in System Settings > Notifications — banner-style notifications auto-dismiss and hide the action buttons the approval cards depend on; quote the §7 ⚑ rule that every grant must be exercised under launchd (`launchctl kickstart -k gui/$(id -u)/com.kahya.kahyad`), not from a terminal; do the grants by hand **now, during the day**. Add the **Full Disk Access** section (§7 table row 3): System Settings > Privacy & Security > Full Disk Access → `+` → add the built, codesigned `kahyad` binary (TCC keys off the responsible process's code identity — W0-04's stable `Kahya Dev` signature is what makes the grant survive rebuilds); restart via `launchctl kickstart -k gui/$(id -u)/com.kahya.kahyad`; verify with an `fs_read` of a TCC-protected directory (e.g. `~/Library/Mail`) issued through a task — success returns bytes, a missing grant returns W3-03's `Tam Disk Erişimi gerekli: <path>`. Document blocked-user handling: until the user clicks the grant, the fs tool fails cleanly with that exact message and the depending work blocks per README (`Status: blocked-user`) — never weaken to a permissive fallback.
10. Write the Go tests (see acceptance) and run `make test && make lint`.
